# 03 UE5 整合方案

## 整合路线总览

```
Houdini HDA
    ├── 路线①  Houdini Engine for Unreal（官方插件）     ← 主流推荐
    ├── 路线②  Houdini Engine + PCG 扩展版               ← 大规模场景
    ├── 路线③  静态导出（FBX/OBJ/Heightmap）             ← 离线烘焙
    └── 路线④  Cesium for Unreal（绕过Houdini）          ← 快速预览
```

---

## 路线① Houdini Engine for Unreal（官方插件）

### 概述

通过此插件，Houdini HDA 可直接作为 UE5 Actor 实例化，**在 UE 编辑器内实时调参**，无需每次手动导出。

### 安装

- 仓库：https://github.com/sideeffects/HoudiniEngineForUnreal
- 支持版本：当前支持 **UE 5.3 / 5.4**（最新两个版本）
- 安装位置：`[ProjectRoot]/Plugins/HoudiniEngine/`

### 核心工作流

```
1. 在 Houdini 中完成 HDA，保存为 .hda 文件
2. 在 UE5 Content Browser 中导入 .hda
3. 拖入场景 → 自动生成 Houdini Asset Actor
4. 在 Details 面板调整暴露参数 → 实时 Cook 并更新场景
5. 满意后使用 "Bake" 按钮将结果烘焙为普通 UE Static Mesh
```

### 输出能力对照表

| 功能 | 支持状态 |
|------|---------|
| Static Mesh 生成 | ✅ 完整支持 |
| Landscape 生成 | ✅ 完整支持 |
| Instanced Static Mesh | ✅ 完整支持 |
| Spline / Curve | ✅ 支持 |
| Nanite Mesh | ✅ UE5.1+ |
| Lumen 兼容 | ✅ 自动兼容 |
| World Partition | ✅ UE5.2+ |
| PCG 节点输出 | ⚠️ 需额外 PCG 版插件 |

## 路线② Houdini Engine + PCG 版（大规模程序化场景）

### 适用场景

- 超大地图（1km² 以上）
- 需要运行时程序化植被/道具散布
- UE5 PCG 图表与 Houdini 逻辑混合

### 插件位置

```
[Houdini安装目录]\engine\unreal\5.X-PCG\
```

### PCG 集成要点

- Houdini HDA 的 Point 输出直接映射为 PCG Point Data
- 在 PCG Graph 中插入 `Houdini Generator` 节点调用 HDA
- 支持 **World Partition + PCG** 的无缝大世界生成

## 路线③ 静态导出（离线烘焙）

适合不需要实时迭代的制作阶段。

### 几何体导出

```
Houdini → File → Export → .fbx（选择三角化 + 法线烘焙）
UE5 → Import → Static Mesh → 调整导入设置（Scale 1.0，坐标系选Y-Forward）
```

### 地形 Heightmap 导出

```
Houdini:
  Height Field Output SOP → 导出 16-bit PNG（高度图）+ RGBA PNG（权重图）

UE5:
  Landscape 工具 → Import → 选择 .png 文件
  → 设置 Scale Z（根据实际高差：Scale Z = 高差m × 100 / 512）
```

## 路线④ Cesium for Unreal（绕过 Houdini 快速预览）

### 适用场景

- 不需要自定义几何体，只需真实卫星建模效果
- 城市规划/地理可视化展示
- 配合 Houdini 生成内容叠加在真实底图上

### 接入步骤

```
1. 安装 Cesium for Unreal 插件（Marketplace 免费）
2. 在 cesium.com/ion 注册获取 Token
3. UE5 → Cesium → 添加 "Cesium World Terrain"
4. 添加 "Google Photorealistic 3D Tiles"（需 Google API Key）
5. 结合 Georeferencing 将自定义 Houdini 内容叠加到真实坐标上
```

### 注意事项

- Google Photorealistic 3D Tiles **按量计费**，商业项目注意成本
- 必须显示 Google 版权信息（`Show Credits On Screen = true`）
- 流式数据不可导出，不能用于离线渲染/打包发布（受 ToS 限制）

## 坐标系与单位对齐备忘

| 引擎/软件 | 坐标系 | 单位 |
|----------|-------|------|
| Houdini | Y-up, Right-hand | 1 unit = 可自定义（建议 1m） |
| Unreal Engine 5 | Z-up, Left-hand | 1 unit = 1 cm |
| OSM/GIS | WGS-84 经纬度 | 度 / 米 |
| Cesium | ECEF + WGS-84 | 米 |

**关键换算：** Houdini 中 1 unit = 1m 时，导入 UE5 需 ×100（或在 HDA 输出时 `transform scale 100`）
