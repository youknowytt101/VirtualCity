# 05 虚拟城市主流技术方案（2024~2025）

## 行业分类

虚拟城市按用途分为两大阵营，技术路线差异显著：

| 阵营 | 代表场景 | 核心需求 | 主流技术栈 |
|------|---------|---------|-----------|
| **游戏 / 影视** | AAA 游戏、影视 VFX、XR 体验 | 艺术可控、高性能渲染、可游玩 | Houdini + UE5 |
| **数字孪生 / 智慧城市** | 城市规划、政务可视化、仿真 | 真实地理数据、实时更新、IoT对接 | GIS + Cesium/CityEngine + UE5/Omniverse |

---

## 一、游戏/影视级：Houdini + UE5 城市样本范式（当前 AAA 标准）

### 1.1 标杆案例：The Matrix Awakens & UE5 City Sample

Epic Games 2021 年发布的《黑客帝国：觉醒》技术演示，确立了当前业界最高标准：

> "7000 栋建筑由数千个模块化构件拼合而成，45,073 辆停放的汽车，建筑布局和细节**全部由 Houdini 生成**，并通过 Houdini Engine 推送进 UE5。"

**City Sample 开放项目**（免费下载，含全套资产和 HDA）：  
https://www.unrealengine.com/marketplace/city-sample

### 1.2 核心技术架构

```
                    ┌─────────────────────────────────────────┐
                    │         Houdini（离线程序化生成）           │
                    │                                         │
  城市规划图（道路网）  →  道路 SOP → 街区划分 → 地块生成          │
  参数（容积率/风格）  →  建筑体量 → 立面模块装配 → 楼顶细节       │
                    │                                         │
                    │   输出 HDA（含建筑点云 + 属性）            │
                    └──────────────┬──────────────────────────┘
                                   │ Houdini Engine for Unreal
                    ┌──────────────▼──────────────────────────┐
                    │         Unreal Engine 5                 │
                    │                                         │
                    │  Nanite    ← 无限面数建筑细节             │
                    │  Lumen     ← 全动态全局光照               │
                    │  PCG       ← 植被/道具/人群程序化填充      │
                    │  World Partition ← 无缝超大地图流送       │
                    │  Chaos     ← 物理破坏/车辆模拟            │
                    └─────────────────────────────────────────┘
```

### 1.3 建筑生成的两种主流策略

#### 策略 A：模块化拼装（主流 AAA）

- 预先制作大量建筑模块（底层/中层/顶层/立面单元）
- Houdini 根据地块大小和风格参数**自动拼装**
- 每个模块支持 Nanite，实例化渲染（ISM/HISM）
- 代表：City Sample、《看门狗》系列

#### 策略 B：程序化 Facade 生成（规则语法）

- 使用形状语法（CGA Rules）或 Houdini VEX 驱动窗户/阳台/装饰细节
- 建筑外观无限变化，不依赖大量手工模块
- 代表：ArcGIS CityEngine（偏 GIS 领域）

### 1.4 独立/中小团队方案

不具备 AAA 管线时的替代路线：

| 工具 | 类型 | 说明 |
|------|------|------|
| **Houdini Indie** | 程序化 | 官方授权 $269/年，功能与 Commercial 几乎一致 |
| **PCG（UE5 内置）** | 程序化散布 | UE5.2+ 内置，无需额外费用，适合植被/小道具 |
| **CityEngine for UE（Esri）** | 规则建筑 | 免费 UE5 插件，直接在引擎内用 CGA 规则生成建筑 |
| **StreetMap Plugin** | OSM 快速导入 | 直接在 UE5 内导入 .osm 生成简单道路和建筑体量 |

## 二、数字孪生 / 智慧城市：GIS + 实时引擎

### 2.1 核心差异

数字孪生城市的诉求与游戏城市完全不同：
- **必须基于真实地理坐标**（WGS-84 / CGCS2000）
- **数据实时更新**（IoT 传感器、交通流、气象）
- **与政府 GIS 平台对接**（ArcGIS、SuperMap、天地图）
- 渲染效果次于游戏，但**数据准确性优先**

### 2.2 主流平台矩阵

```
数据层
  ├── Esri ArcGIS Platform       → 权威 GIS 数据、空间分析
  ├── Cesium ion                 → 3D Tiles 数据托管与流送
  ├── Bentley iTwin              → 基础设施 BIM + GIS 融合
  └── 国内：超图 SuperMap / 中地数码

渲染层
  ├── Unreal Engine 5            → 最高画质，游戏引擎级交互
  │     ├── Cesium for Unreal    ← 地理数据接入
  │     └── ArcGIS Maps SDK      ← Esri 数据直连
  ├── NVIDIA Omniverse           → 多软件协作、物理仿真
  └── Unity + Cesium             → 更轻量，适合 Web/移动端

IoT/实时数据层
  └── Kafka / MQTT → 实时数据流 → 引擎内 Blueprint/材质参数驱动
```

### 2.3 Cesium for Unreal（数字孪生首选接入方案）

- **功能**：在 UE5 内直接流式加载真实世界 3D 瓦片（包括 Google 实景 3D）
- **地理定位**：支持精确的 WGS-84 坐标系，确保自定义模型与真实世界对齐
- **配合使用**：  
  - `ArcGIS Maps SDK for Unreal` → 实时叠加 GIS 矢量要素层  
  - 自定义 Houdini HDA → 在真实底图上叠加规划方案

### 2.4 NVIDIA Omniverse（多平台协作场景）

- 基于 **OpenUSD** 格式，各软件（Revit/3dsMax/Houdini/Maya）实时同步
- Bentley iTwin + Omniverse = 基础设施数字孪生事实标准之一
- 适合**多专业协同**（建筑/结构/机电同步）而非单一美术管线

## 三、新兴方向：AI 辅助城市生成（2024~2025 前沿）

### 3.1 Proc-GS（SIGGRAPH 2024）

- 用 **3D Gaussian Splatting** 生成程序化建筑，无需建模
- 从少量参考照片/描述 → 生成可编辑的程序化 3D 城市建筑
- 目前仍在学术阶段，尚未进入主流制作管线
- 论文：https://arxiv.org/html/2412.07660v1

### 3.2 NeRF / 3DGS 扫描重建

- 使用无人机航拍 + NeRF/3DGS 重建真实城市区块
- 与传统 Houdini 管线配合：扫描重建提供底图，Houdini 做程序化填充
- 工具：**Luma AI**、**Polycam**、**Reality Capture**、**Metashape**

### 3.3 生成式 AI（现阶段局限）

- AI 目前能做：概念图生成、纹理生成、单体建筑参考
- AI 现阶段不能替代：大规模拓扑一致的城市程序化生成
- 趋势：AI 生成的素材库 → 人工拼装到 Houdini/UE5 管线中

## 四、选型建议

| 项目类型 | 推荐路线 | 关键工具 |
|---------|---------|---------|
| AAA 游戏开放世界城市 | Houdini 全程序化 + UE5 City Sample 范式 | Houdini + Houdini Engine + UE5 Nanite/Lumen/PCG |
| 影视/XR 虚拟城市 | 同上，强调镜头美学 | 同上 + Cesium 实景底图参考 |
| 独立游戏/小团队 | CityEngine for UE + PCG + 手工补充 | UE5 PCG + CityEngine UE 插件 + OSM |
| 政务数字孪生/智慧城市 | GIS 数据 + Cesium + ArcGIS SDK | Cesium for UE + ArcGIS Maps SDK + IoT 数据层 |
| 城市规划可视化 | CityEngine + UE5 / Omniverse | ArcGIS CityEngine + Cesium |
| 快速原型/预研 | OSM 直导 UE5 | StreetMap Plugin / Cesium 3D Tiles |
