# 数据处理自动化

用于预留地图数据相关的非官方自动化脚本位置。

## MVP 阶段结论

第一阶段不建议先开发数据处理脚本。

更现实的做法：

- 手动从 Overpass Turbo 框选并导出 `.osm`。
- 手动记录 bbox、城市名、数据来源和下载时间。
- DEM 可以先不接入，或只手动准备一份测试数据。
- 坐标问题先在 Houdini 内通过参考点和比例检查解决。

## 后续何时需要脚本

只有出现以下情况，才考虑开发：

| 触发条件 | 再考虑的工具 |
|---|---|
| 经常要下载多个区域 | OSM bbox 下载脚本 |
| 多个数据源经常错位 | 坐标对齐检查脚本 |
| DEM 反复裁切重采样 | DEM 裁切脚本 |
| 需要批量记录数据版本 | 区域配置 / 数据版本记录脚本 |

当前目录先只作为预留位。
