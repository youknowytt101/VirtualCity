# 10 AI 启动自检清单

> 本文件用于 AI 每次打开 VirtualCity 项目时快速自检。  
> 目标：避免漏读状态、重复造文档、跳过当前阶段或误推进复杂内容。

---

## 1. 必读入口检查

开始任何任务前，确认已读取：

- [ ] `项目管理/00_AI接手指南.md`
- [ ] `项目管理/02_当前状态与下一步.md`
- [ ] `项目管理/01_资料地图.md`
- [ ] `项目管理/08_任务看板.md`
- [ ] `项目管理/09_决策记录.md`
- [ ] `项目管理/project_manifest.json`

---

## 2. 当前阶段判断

根据入口文件判断当前阶段：

- [ ] `mvp_preparation`：准备区域、数据、配置。
- [ ] `mvp_houdini`：建立 Houdini MVP 网络。
- [ ] `mvp_ue5`：导入 UE5 并 Bake。
- [ ] `pipeline_standardization`：固化流程和自动化。
- [ ] `scale_up`：扩展到 1km²、Tile、World Partition。

当前默认阶段：**pipeline_standardization**。

---

## 3. 缺失项检查

按顺序检查，找到第一个缺口后优先处理：

| 顺序 | 检查项 | 当前期望 |
|---:|---|---|
| 1 | 是否有 MVP 区域 bbox | 尚未有，需用户确定 |
| 2 | 是否有区域记录 | `项目管理/区域记录/{area_id}.md` |
| 3 | 是否有区域配置 | `Config/{area_id}.area.json` |
| 4 | 是否有 OSM 文件 | `RawData/OSM/{area_id}_osm_v001.osm` |
| 5 | 是否有 Houdini `.hip` | `Houdini/Hip/VC_{area_id}_citygen_v001.hip` |
| 6 | 是否有 HDA | `Houdini/HDA/VC_{area_id}_cityblock_v001.hda` |
| 7 | 是否有 UE5 项目 | `UE5/VirtualCityUE/VirtualCityUE.uproject` |
| 8 | 是否完成 Bake | UE5 中有 Bake 后资产和关卡 |

---

## 4. 禁止误推进检查

如果当前仍是 MVP 前准备阶段，不要主动推进：

- [ ] City Sample 级完整管线。
- [ ] Mass AI。
- [ ] 大型自研插件。
- [ ] 全国或整城级地图。
- [ ] World Partition 自动分块。
- [ ] PDG 大规模批处理。

除非用户明确要求。

---

## 5. 每次任务结束检查

结束前确认：

- [ ] 是否更新 `02_当前状态与下一步.md`。
- [ ] 是否更新 `03_迭代日志.md`。
- [ ] 是否需要更新 `08_任务看板.md`。
- [ ] 是否需要更新 `09_决策记录.md`。
- [ ] 是否需要更新 `project_manifest.json`。
- [ ] 是否需要更新 `document_index.json`。

---

## 6. 用户只说“继续”时的默认动作

默认按以下顺序推进：

1. 完善项目接手/索引/状态体系。
2. 检查任务看板中 P0 任务。
3. 如果文档体系已完整，推进当前 active_area 的下一个缺口。
4. 不在没有 bbox 的情况下虚构区域数据。

---

## 7. 用户提供地图框选结果时的默认动作

当用户发送以下任意内容时，视为区域接入请求，AI 应直接执行，**不得把整理工作再返还给用户**：

- 一组经纬度数值
- `bbox: west,south,east,north` 格式文本
- GeoJSON Polygon / Feature
- 地图截图 + 地点名称
- bboxfinder / Overpass Turbo / Geoman 复制结果

AI 必须完成：

- [ ] 生成或确认 `area_id`
- [ ] 创建 `项目管理/区域记录/{area_id}.md`
- [ ] 创建 `Config/{area_id}.area.json`
- [ ] 创建 `Config/{area_id}.overpass.txt`（含填好 bbox 的查询语句）
- [ ] 更新 `项目管理/02_当前状态与下一步.md`
- [ ] 更新 `项目管理/08_任务看板.md`
- [ ] 更新 `项目管理/project_manifest.json` 的 `active_area`
- [ ] 追加 `项目管理/03_迭代日志.md`

如果 bbox 值不完整，区域状态标记为 `bbox_pending`，不得伪造坐标。  
如果 bbox 完整，状态设为 `ready_for_osm_download` 并在输出中给出可直接使用的 Overpass 查询。

---

## 8. 完整自动化流程测试入口

当用户要求“完整流程测试 / 重新测试 / 从头跑一遍 / 测试自动化管线 / 从网页框选开始”时，默认入口必须是：

```bash
uv run python area_picker.py
```

工作目录：

```text
D:\VirtualCity\Scripts
```

AI 应等待用户在网页中框选区域并点击“开始生成”，然后监控该进程输出直到 `set_area.py`、`refine_data.py`、Houdini recook 全部结束。

除非用户明确给出 bbox，或明确要求“跳过网页 / 直接用这个 bbox 跑”，否则不要直接执行：

```bash
uv run python set_area.py ...
```

`set_area.py` 是底层命令入口，不是完整用户流程入口。
