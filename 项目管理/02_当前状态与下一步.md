### Houdini 资产质量

- 道路：
  - `road_width_flat` 强制接 `resample_roads`，避免稀疏长线段生成大面片。
  - `road_width_flat` 增加 highway 类型最终 clamp，避免异常 `width/lanes` 标签放大道路面。
  - `road_strips` 当前为 v5，使用全顶点路口处理、调试属性、自交/尖角保护、有限凸包填充和复杂路口降级。
  - `snap_road_strips` 对道路条带所有顶点二次贴地，避免坡地侧边埋入地形。
  - `road_bbox_clip` 会清理裁剪后近重复点，对 5+ 边形做安全 fan 三角化，并跳过过尖边界碎片。
  - `road_clipped_faces` 已纳入 Model QA，检查裁剪后的道路面状态。
- 地形：
  - `dem_terrain` 后接 `dem_subdivide`，Bilinear iterations=2，等效约 7.5m 网格，用于道路和建筑吸附。
- 建筑：
  - `snap_bld_to_terrain` 使用逐顶点 MAX 地形高度，优先避免坡地建筑被埋。
  - 新增 `bld_foundation` / `bld_with_foundation`，用建筑最终 body 底边生成裙边，解决坡地悬空。
  - 裙边颜色与建筑 body 保持一致，并用 `is_foundation` 标记做 QA。
  - 新增 `bld_footprint_bevel`，只处理 footprint 外角；当前规则为外角角度 `<=100°`，带 `2°` 容差。

### 自动化 QA

新增 `Scripts/houdini_model_qa.py`，当前 quick 模式检查：

- 必需节点存在且非空。
- 地形是否使用加密后的 `dem_subdivide`。
- 建筑 body / foundation / final 颜色一致。
- footprint bevel 是否真的产生且无过短边。
- 建筑最终法线是否有效。
- `is_foundation` tag 数量是否匹配。
- foundation side normal 是否与 building side normal 对齐。
- foundation top edge 是否与 building bottom edge 对齐。
- 建筑是否低于地形阈值。
- road strip 是否存在异常大面片、自交、尖角、长宽比异常。
- road_clipped 是否存在异常大面片、自交、尖角、长宽比异常，并确认裁剪后 n-gon 已规整。
- road_clipped 是否低于地形阈值。

QA 已改为在 Houdini 进程内部 bundle 计算大批量几何统计，避免 RPYC 对每点/每面远程遍历导致超时。

## 4. 当前正确测试方式

当用户说“重新测试 / 从头测试 / 全流程测试 / 测试自动化管线”时，必须运行：

```bash
cd Scripts
uv run python area_picker.py
```

然后从网页入口触发区域选择和流程执行。完整流程包括：

1. 弹出 Leaflet 网页。
2. 用户框选或确认 bbox。
3. 获取 / 恢复 OSM、FABDEM、Overture 数据。
4. 数据清洗与数据 QA。
5. Houdini recook。
6. Model QA quick。
7. 写入 `Config/houdini_build_status.json`。

不得把直接运行 `_recook_new_area.py` 或 `set_area.py` 称为“全流程测试”。只有用户明确要求“当前区域重构 / 跳过网页 / 直接用这个 bbox”时，才使用底层入口。

## 5. 当前主要风险与处理方式

| 风险 | 当前处理 |
|---|---|
| Houdini RPYC 端口 18811 被占用 | `pythonrc.py` 需避免重复启动 server；端口占用时先确认旧 Houdini 进程 |
| QA 远程遍历几何过慢 | 在 Houdini 内部执行 bundle helper，再一次性返回 JSON |
| Houdini 节点重建后旧 proxy 失效 | `_recook_new_area.py` 验证阶段重新取节点句柄并重试 |
| area_picker 重复启动导致日志串台 | `/run` 已加 running 防重入；状态文件必须同一区域才算成功 |
| 网页自动关闭不可靠 | 浏览器可能阻止 `window.close()`；后续应改为完成态页面，而不是依赖自动关闭 |
| 建筑裙边与 body 错位 | foundation 从最终 `bld_clipped` body 底边生成，不从早期 footprint 生成 |
| 道路坡地侧边埋入地形 | road strip 全顶点二次贴地 |
| DEM 原始网格过稀 | 使用 `dem_subdivide` 作为所有吸附节点的 terrain input |

## 6. 下一步优先级

### P0：人工审核当前 Houdini 输出

在 Houdini 中查看 `OUT_city`，重点审核：

- 道路在山地/丘陵区域是否仍有局部埋地、悬空或断裂。
- 建筑裙边是否视觉合理，不错位、不反色、不穿插。
- 建筑倒角是否只出现在外角，且不会制造过短边或奇怪斜面。
- 俯视角整体城市关系是否可信：道路宽度、建筑密度、地形比例。

### P0：修复 area_picker 完成态体验

当前后台流程完成后，浏览器标签页不一定自动关闭。更合理的方向：

- 页面加载后始终轮询 `/status`。
- 后台已 running/done 时自动切进进度或完成态。
- 完成后显示“已完成，可关闭”，不要依赖 `window.close()`。

### P1：继续 Houdini 资产质量迭代

- 道路交叉口形态继续优化。
- 建筑 skirt/foundation 视觉厚度、材质和底边处理继续调参。
- 评估是否需要道路边缘/路缘石/人行道层级。
- 记录晋级为基准样区的区域，不必为每个快速测试区写完整区域文档。

### P2：Houdini 确认后再恢复 UE5 输出链

当前 UE5 Export / Import 不作为默认测试流程。只有 Houdini 输出经人工审核通过后，再运行：

```bash
cd Scripts
uv run python export_and_import.py
```
