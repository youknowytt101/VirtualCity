# 07 MVP QA 检查清单

> 本文件用于检查第一个城市 MVP 区块是否真的跑通。  
> 对应机器可读模板：`配置/qa_checklist.template.json`。

---

## 1. 数据检查

| 检查项 | 通过标准 | 状态 |
|---|---|---|
| bbox 已记录 | 有 west / south / east / north | 未检查 |
| OSM 文件存在 | `原始数据/OSM/` 下有对应 `.osm` | 未检查 |
| DEM 文件状态明确 | 有 DEM 或明确本轮暂不使用 | 未检查 |
| 数据来源记录 | 记录来源和下载时间 | 未检查 |
| 坐标系明确 | WGS84 / GCJ-02 / BD-09 / CGCS2000 / 待转换 | 未检查 |

---

## 2. Houdini 检查

| 检查项 | 通过标准 | 状态 |
|---|---|---|
| OSM 可导入 | Houdini 能读取数据 | 未检查 |
| 道路可见 | 主路和支路能生成路面 | 未检查 |
| 建筑可见 | 建筑轮廓能拉伸成体量 | 未检查 |
| 高度规则可调 | 默认楼层和高度倍率可调 | 未检查 |
| 输出节点清晰 | 有 `OUT_buildings` / `OUT_roads` 等 | 未检查 |
| HDA 可保存 | 存在 `.hda` 或 `.hdalc` | 未检查 |

---

## 3. UE5 检查

| 检查项 | 通过标准 | 状态 |
|---|---|---|
| UE5 项目存在 | 存在 `.uproject` | 未检查 |
| Houdini Engine 可用 | 插件启用且能导入 HDA | 未检查 |
| 比例正确 | 人、车、道路、建筑尺度合理 | 未检查 |
| Bake 成功 | Bake 后可脱离 Houdini 查看 | 未检查 |
| Play 正常 | 无严重报错 | 未检查 |
| 基础展示 | 至少 3 个截图视角 | 未检查 |

---

## 4. 文档检查

| 检查项 | 通过标准 | 状态 |
|---|---|---|
| 区域记录已更新 | `项目管理/区域记录/{area_id}.md` 完整 | 未检查 |
| 当前状态已更新 | `02_当前状态与下一步.md` 反映真实状态 | 未检查 |
| 迭代日志已更新 | `03_迭代日志.md` 记录本轮变更 | 未检查 |
| manifest 已更新 | 如路径或阶段变化，JSON 已同步 | 未检查 |

---

## 5. MVP 通过定义

全部核心项通过后，MVP 才算跑通：

- OSM 道路导入成功。
- 建筑轮廓生成体量。
- Houdini 输出 HDA 或静态资产。
- UE5 成功导入并 Bake。
- 场景可 Play。
- 文档和区域记录完整。
