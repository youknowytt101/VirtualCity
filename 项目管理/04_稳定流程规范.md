# 04 稳定流程规范

> 本文件用于把 VirtualCity 从一次性调研项目，逐步固化为可复用、可自动迭代的生产流程。  
> 原则：先人工跑通，再记录参数，最后把高频重复步骤自动化。

---

## 1. 稳定流程总览

目标流程：

```text
区域选择
    ↓
数据获取
    ↓
GIS 预处理
    ↓
Houdini 程序化生成
    ↓
HDA / 静态资产输出
    ↓
UE5 导入 / Bake
    ↓
展示与性能检查
    ↓
问题记录 / 参数固化 / 下一轮迭代
```

---

## 2. 每轮迭代必须记录的内容

每轮生成一个城市区域，都要记录：

| 类型 | 必填内容 |
|---|---|
| 区域信息 | 区域名、城市、bbox、面积、坐标系 |
| 数据信息 | OSM 文件、DEM 文件、数据来源、下载时间 |
| Houdini 信息 | `.hip` 路径、HDA 路径、关键参数、随机种子 |
| UE5 信息 | 关卡路径、Bake 输出、Data Layer、插件版本 |
| 结果信息 | 截图、问题、性能、是否可复现 |
| 下一步 | 需要修复或增强的内容 |

建议每个区域建立独立记录：

```text
项目管理/区域记录/mvp_area.md
```

## 3. 阶段门禁

### Gate 0：环境可用

进入数据阶段前必须满足：

- Houdini 可运行。
- SideFX Labs 可用。
- UE5 可运行。
- Houdini Engine for Unreal 可启用。

### Gate 1：数据可复现

进入 Houdini 阶段前必须满足：

- bbox 已记录。
- OSM 文件已保存。
- 数据来源已记录。
- 坐标系已确认或标记为待确认。

### Gate 2：Houdini 可生成

进入 UE5 阶段前必须满足：

- 道路、建筑体量可见。
- 道路和建筑没有明显错位。
- 关键参数可调。
- HDA 或静态导出文件存在。

### Gate 3：UE5 可 Bake

进入展示阶段前必须满足：

- HDA 可导入 UE5。
- 生成结果比例正确。
- Bake 后可脱离 Houdini 查看。
- Play 模式无严重报错。

### Gate 4：流程可复用

进入更大范围或自动化阶段前必须满足：

- 换一个 bbox 不需要重写整套流程。
- 参数和版本已记录。
- 本轮问题已写入迭代日志。

## 5. 单位与坐标规范

推荐统一约定：

```text
Houdini 内部：1 unit = 1 meter
UE5 内部：1 Unreal unit = 1 centimeter
Houdini → UE5：统一乘以 100
GIS 原始数据：先转换到项目局部米制坐标
```

不要在多个节点中分散缩放，缩放应集中在 HDA 输出端或导入配置中管理。

---

## 6. 自动化触发条件

只有满足以下条件之一，才考虑开发脚本：

- 同一步骤重复执行 3 次以上。
- 手动操作已经导致错误或遗漏。
- 需要批量处理多个区域或 Tile。
- 手动记录参数已经影响可复现性。

自动化优先级：

1. 区域配置和 metadata 记录。
2. OSM / DEM 文件路径和 bbox 检查。
3. 坐标系和单位检查。
4. Houdini 批量 Cook。
5. UE5 资产整理和 Data Layer 分配。
6. 大地图 Tile / World Partition 自动化。

---

## 7. 每轮 QA Checklist

| 检查项 | 通过标准 |
|---|---|
| bbox 记录 | 可复现下载同一区域 |
| 坐标系 | 已确认或明确标记待转换 |
| 道路连续性 | 没有明显断裂和错位 |
| 建筑位置 | 不压主路，不大面积偏移 |
| 建筑高度 | 没有异常高楼或全为同高 |
| 地形比例 | 高差合理，不悬空 |
| UE 比例 | 人、车、路、楼尺度正常 |
| Bake 结果 | 脱离 Houdini 可打开 |
| 性能 | 小区块可顺畅浏览 |
| 文档 | 状态和日志已更新 |
