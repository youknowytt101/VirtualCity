## 2026-05-28｜Houdini 资产质量迭代、Model QA 与全流程测试固化

### 完成内容

- 明确当前工程阶段：最终目标是俯视角虚拟城市构建，当前处于“数据获取 → 数据清洗 → Houdini 自动化构建”的快速实验迭代阶段，不是最终 UE5 整合输出阶段。
- 道路链路继续优化：
  - 修复 `road_width_flat` 可能未强制接入 `resample_roads` 的风险，避免稀疏中心线生成大面积错误道路面片。
  - `road_strips` 继续使用全顶点路口处理、交叉插点和凸包填充。
  - 新增道路条带全顶点二次贴地，解决山地/丘陵地区道路侧边埋入地形的问题。
- 地形链路优化：
  - 在 `dem_terrain` 后加入 `dem_subdivide`，Bilinear iterations=2，将 30m DEM 视觉吸附目标加密到约 7.5m。
  - 下游道路和建筑吸附统一使用 `dem_subdivide`。
- 建筑链路优化：
  - `snap_bld_to_terrain` 改为逐顶点 MAX 地形高度，优先避免坡面建筑被埋。
  - 新增并修复建筑 foundation/skirt：从最终建筑 body 底边生成，颜色与 body 一致，使用 `is_foundation` 标记。
  - 新增 footprint 倒角逻辑，只处理外角；当前阈值为 `<=100°`，带 `2°` 容差。
- 新增 `Scripts/houdini_model_qa.py`，形成 Houdini Model QA quick：
  - required nodes
  - terrain density
  - building/foundation color
  - footprint bevel
  - building normals
  - foundation tags / normals / alignment
  - building terrain fit
  - road faces
  - road terrain fit
- 将 QA 中大批量 Houdini 几何统计迁移到 Houdini 进程内部 bundle helper，避免 RPYC 逐点/逐面遍历超时。
- `_recook_new_area.py` 增加 stale Houdini node proxy 重试逻辑，避免节点重建后 `ObjectWasDeleted` 中断流程。
- `area_picker.py` 增加 running 防重入，并要求同一区域 Houdini 状态完成才算成功，避免重复启动导致日志串台或假完成。
- 使用多个区域完成完整流程验证：
  - `area_12.917_100.868`：数据 QA 8 pass / 1 warn / 0 fail；Model QA 11 pass / 0 warn / 0 fail。
  - `area_12.918_100.865`：数据 QA 8 pass / 1 warn / 0 fail；Model QA 11 pass / 0 warn / 0 fail。
- 本地 `main` 已推送到 GitHub `origin/main`，最新提交为 `9e9f436 Add area 12.918 100.865 pipeline test`。

### 决策

- 用户说“重新测试 / 从头测试 / 全流程测试”时，必须从 `area_picker.py` 网页入口开始，到 Houdini build status 与 Model QA 通过为止。

- 当前 UE5 Export / Import 仍在人工审核 Houdini `OUT_city` 之后触发，不并入默认测试。
- Model QA 是快速实验阶段的必要护栏，但不替代人工视口审核。
- 快速测试区不必全部写完整区域记录；只有晋级为基准样区或展示样区的区域才补详细区域文档。
